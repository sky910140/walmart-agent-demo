# 审查整改说明

这份说明逐项回应《全面审查报告》。结论不是“全部照单全收”：面试项目的优先级应是可运行、可解释、可验证。以下明确区分已经实现、已文档化但需要外部网络完成的数据、以及有意延期的功能。

| 审查项 | 决策 | 实现或理由 |
| --- | --- | --- |
| 1. 缺上证/主要指数 | 采纳 | 新增 `download-markets` 和 `MAJOR_A_SHARE_INDEXES`：沪深 300 `sh000300`、上证综指 `sh000001`、深证成指 `sz399001`。三组 20 年 CSV 与 `.meta.json` 均已真实下载、验证并随源码提交。 |
| 2. Demo 太少 | 采纳 | `DEMO_OUTPUTS.md` 补充 Web、偏好记忆、JSON、trace、跨公司、三指数入口、验证守卫和 BM25 失败模式。所有示例注明是真实运行还是确定性测试。 |
| 3. 多模型协同价值不够具体 | 采纳 | 规划输出现已受限地扩展检索 query；新增集成测试证明泛化问题因规划词命中流动性证据，且无引用 Doubao 草稿被 DeepSeek verifier 的 `[S1]` 结果替代。文档不虚构两个模型的质量差值。 |
| 4. 没展示 BM25 局限 | 采纳 | `DESIGN.md`、`DEMO_OUTPUTS.md` 和中文文件说明均保留 `top-line pressure` 对 `revenue decline` 的词法失配案例，并说明向量/混合检索是后续优先项。 |
| 5. 中文 tokenizer | 采纳 | 连续中文改为重叠 bigram，如 `上海证券交易所` 生成 `上海、海证、证券、券交、交易、易所`；有回归测试。 |
| 6. XBRL 噪声 chunk | 采纳 | 解析器跳过 inline-XBRL 头部/隐藏/资源/context/unit，索引前按 schema/context 噪声密度过滤；不会粗暴删除正文中正常提及 XBRL 的内容。 |
| 7. manifest 覆盖 | 采纳 | 下载前读取并校验旧 manifest，按 `document_id` 去重合并，保留不同历史记录；损坏行显式报错。 |
| 8. CLI traceback | 采纳 | `main()` 捕获可预期 I/O、参数、JSON、运行时异常，输出 `Error: ...` 和退出码 2；新增测试。 |
| 9. 市场错误被吞 | 采纳 | `AgentResponse.warnings` 与 Markdown 的 `Data warnings` 显式呈现，并使用标准 `logging` warning 记录，不影响已有 filing 证据回答。 |
| 10. HTML 输出 | 延期 | Markdown 已能表达表格和引用，JSON 满足机器接口。HTML 不解决本题核心风险，额外渲染层会引入转义/XSS 责任，因此保留为后续工作。 |
| 11. 偏好表达覆盖 | 部分采纳 | 增加 `I'm interested in`、`I am interested in`、`感兴趣`；仍维持六类白名单和显式意图门槛，避免把任意问题永久记忆。 |
| 12. 无 logging | 采纳（最小化） | 增加模块级 logger，用于市场数据不可用等非致命事件；用户面同时有 warning。未引入复杂日志基础设施，避免输出请求内容或密钥。 |
| 13. 测试不足 | 采纳 | 从 11 项扩展到 26 项，增加坏 CSV、CLI、UTF-8、manifest 保留、规划/核验协同、市场证据优先级、XBRL、中文、三指数与 warning 的正负路径、`verify-models` 的成功/失败退出码，以及本地 `.env` 存在时的测试隔离。`coverage` 保留为可选开发依赖；本机未依赖其完成运行测试。 |
| 14. 未使用 import | 采纳 | 移除 `agent.py` 的未使用 `asdict`。 |

## 为什么不全量重做

没有把 BM25 替换为向量数据库、没有接入股票级全市场数据、没有加入 HTML/PPT 或多用户数据库。这些都可能扩充功能面，却会增加服务依赖、数据授权、环境配置和无法现场解释的状态。当前整改优先处理了会影响正确性、可追溯性、演示可信度和 Windows 可运行性的项。

## 面试时应主动说明的事实

1. 两个主力模型只在配置密钥时远程调用；无密钥时，离线提取回答是刻意的安全降级，不是模拟模型输出。`verify-models` 以固定 `READY` prompt 验证两家 provider，不发送金融数据；2026-07-11 已在本地 `.env` 配置后成功验证 `doubao-seed-evolving` 和 `deepseek-v4-pro`。密钥本身未被读取、显示或提交。
2. verifier 的有效性来自“只允许现有 `[S#]`”的程序守卫；它不是对财务事实的独立外部审计。
3. 规划器影响的是检索候选词，不是证据本身。证据仍必须来自本地 source chunk 或显式标记的 Web snippet。
4. 当前真实下载验证了 10 家最新 10-K、CSI 300、上证综指和深证成指。三组市场 CSV、每年请求 URL 和元数据已提交，现场市场 demo 不依赖公网下载。
5. BM25 的透明性是取舍：它便于展示排名和离线运行，但不是语义检索。下一步是有评测集支撑的混合检索，而不是盲目引入向量服务。
